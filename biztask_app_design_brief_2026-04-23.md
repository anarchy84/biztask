# BizTask 모바일 앱 디자인 브리프

> **대상**: Claude Design  
> **작성일**: 2026-04-23  
> **플랫폼**: React Native (iOS + Android 동시 출시)  
> **디자인 결과물**: Figma 시안 + 디자인 토큰 + 컴포넌트 스펙

---

## 0. TL;DR (3줄 요약)

1. 40~60대 소상공인이 "퇴근 후 침대에서 낄낄대는" 커뮤니티 앱이다.
2. 블라인드 스타일 세로 피드 + 좋아요/싫어요 + 한국어 가독성 최우선.
3. 어르신 친화 톤인데 힙하진 않아야 함 (네이버카페 + 블라인드 사이 어딘가).

## 1. 프로젝트 개요

### 1-1. 서비스 정체성
- **서비스명**: BizTask (비즈태스크)
- **도메인**: biztask.kr (현재 웹 운영 중, 앱 신규 출시)
- **한 줄 설명**: "활어처럼 펄떡이는 소상공인 커뮤니티"
- **콘텐츠 비율**: 유머 80% / 비즈니스 Q&A 20%
- **차별점**: 네이버카페의 "소속감" + 블라인드의 "익명 낄낄" + 디시의 "짤 문화"를 섞은 한국형 모바일 커뮤니티

### 1-2. 타겟 유저
- **주 타겟**: 40~60대 소상공인·자영업자·1인 셀러
- **보조 타겟**: 30대 온라인 셀러, 프리랜서

### 1-3. 비즈니스 목적
- 하루 1~2회 열어서 10분 낄낄대고 나가는 "습관 앱"
- 진지한 비즈니스 질문은 별도 카테고리에서 해결
- 광고/제휴 수익 모델은 추후 (지금은 커뮤니티 볼륨 확보)

## 2. 디자인 철학 & 레퍼런스

### 2-1. 지향점 (이런 앱을 참고)

| 앱 | 참고할 것 |
|---|---|
| **블라인드** | 모바일 최적화 세로 피드, 회사(→업종) 배지, 짧은 카드, 공감·댓글 카운트만 표시 |
| **네이버카페** | 한국어 가독성, 어르신 친화 톤, 게시판 친근함 |
| **당근마켓** | 따뜻한 컬러, 큰 터치 타겟, 중앙 플로팅 CTA |
| **디시인사이드 앱** | 짤방/이미지 포스트 친화 레이아웃 |

**메인 레퍼런스는 블라인드**, 나머지는 부분 참고.

### 2-2. 피할 것 (Anti-pattern)

- 레딧식 업/다운보트 점수 경쟁
- 사이드바 / 햄버거 메뉴 남용
- 복잡한 서브카테고리 트리
- 힙한 MZ 톤 (네온·과한 그라데이션·영어 마이크로카피)
- 하얀 배경에 얇은 회색 작은 폰트 (어르신 가독성 망함)
- 팝업 광고식 모달

## 3. 타겟 유저 페르소나

### 페르소나 1. 치킨집 사장 박씨 (55세, 남)
- 프랜차이즈 치킨집 7년차
- 퇴근 후 새벽 1~2시, 침대 누워서 접속
- 진상 손님 썰, 배달 플랫폼 욕, 업계 풍문 좋아함
- 글쓰기보다 **좋아요/댓글**이 편함
- 시력 좋지 않음 → **글자 크고 줄 간격 넓게**

### 페르소나 2. 1인 온라인셀러 김씨 (42세, 여)
- 스마트스토어/쿠팡 셀러 3년차
- 출퇴근 지하철에서 짬짬이 접속
- 광고 ROAS Q&A 진지하게 읽음 + 유머 스레드로 스트레스 풀기
- 스크롤 많이 함 → **빠른 로딩, 무한 스크롤, 이미지 최적화**

### 페르소나 3. 네일샵 원장 이씨 (38세, 여)
- 1인 네일샵 운영
- 손님 없는 시간에 잠깐잠깐
- 인스타/당근 익숙 → 모던한 UI OK
- 이미지 있는 포스트 선호
- 짧게 자주 접속 → **푸시 알림이 재진입 핵심**

## 4. 화면 목록 & 플로우

### 4-1. 홈 피드 (Home) — **가장 중요한 화면**

**구조 (위→아래):**
1. 상단 바: 로고 | 검색 | 알림 (46pt 높이)
2. 카테고리 탭 (가로 스크롤): `전체` `인기` `Q&A` `유머` `짤방` `푸념`
3. 포스트 카드 세로 리스트 (블라인드 스타일)
4. 하단 탭바: `홈` `인기` `✍️쓰기` `알림` `내정보`

**포스트 카드 안에 들어갈 정보:**
- 상단 메타: `닉네임 · 업종배지 · 작성시간` (예: "편의점빌런 · 편의점 · 3분 전")
- 중간: **제목** (최대 2줄, 말줄임)
- 중간: **본문 미리보기** 1~2줄 (텍스트 포스트만)
- 이미지 있으면: 썸네일 (가로 꽉찬 큰 이미지 or 우측 정사각 썸네일 — 제안 부탁)
- 하단 액션: `👍 12` `👎 1` `💬 8`

### 4-2. 글 상세 (Post Detail)
- 상단: 뒤로가기 | 카테고리명 | 더보기(공유/신고)
- 본문 영역:
  - 제목 (크게)
  - 메타 (닉네임/업종배지/시간)
  - 본문 (이미지 인라인 가능)
  - 액션 버튼 행: 좋아요 / 싫어요 / 댓글 / 공유
- 댓글 리스트
  - 평면 + **1단 중첩까지만** (2단 이상 금지)
  - 댓글별: 닉네임·업종·시간 / 내용 / 좋아요·답글
- 하단 고정: 댓글 입력창

### 4-3. 글쓰기 (Write)
- 카테고리 선택 (드롭다운 or 칩)
- 제목 입력
- 본문 입력 (텍스트 + 이미지 첨부 버튼)
- 우상단: 작성 버튼

### 4-4. 인기 피드 (Popular)
- 24시간 / 일주일 / 이달 탭
- 나머지는 홈 피드와 동일 카드 구조

### 4-5. 알림 (Notifications)
- 내 글에 달린 댓글
- 내 글/댓글에 눌린 좋아요
- 내 댓글에 대댓글
- 시간 역순

### 4-6. 내 프로필 (Profile)
- 프로필 헤더: 닉네임 | 업종배지 | 가입일
- 통계: 작성글 수 / 받은 좋아요 / 댓글 수
- 탭: `내가 쓴 글` `내가 쓴 댓글` `좋아요한 글`
- 설정 진입점

### 4-7. 로그인 / 회원가입
- **소셜 로그인 우선**: 카카오 > 네이버 > Apple (iOS)
- 회원가입 추가 정보: **닉네임 + 업종 선택** (업종은 필수)
- 업종 리스트 예: 음식점 / 편의점 / 카페 / 미용 / 온라인셀러 / 학원 / 서비스업 / 기타 (최종 목록은 이 브리프 9절 참고)

## 5. 컴포넌트 스펙

### 5-1. 포스트 카드 (Feed Card)
- 세로 배치, 카드 간 0pt 간격 + 하단 1pt 구분선
- 내부 패딩: 16pt
- 배경: 화이트
- 터치 시 살짝 어두워짐 (pressed state)
- **이미지 없는 카드 / 이미지 있는 카드** 2가지 변형

### 5-2. 좋아요 / 싫어요 버튼
- **좋아요 (Like)**:
  - 아이콘: 엄지 or 하트 (제안 필요)
  - 비활성: 회색 아웃라인
  - 활성: 긍정 컬러 채움 + 카운트 숫자 컬러 변경
  - 더블탭으로도 좋아요 가능 (인스타식)
- **싫어요 (Dislike)**:
  - 아이콘: 엄지 내림 (👎)
  - 비활성: 회색 아웃라인
  - 활성: 회색→진한 그레이 (빨강은 과함 — Claude Design 판단)
  - 싫어요는 **숫자를 크게 강조하지 않음** (부정 행동 유도하지 않기 위해)

### 5-3. 업종 배지 (Industry Badge)
- 블라인드 회사 배지처럼 닉네임 옆에 배치
- 형태: 라운드 사각 (4pt rounding), 배경 옅은 컬러, 텍스트 진한 컬러
- 업종별 컬러 8~10종 팔레트 제안 바람
- 크기: 11~12pt 텍스트

### 5-4. 하단 탭바 (Bottom Tab Bar)
- 5개 탭: `홈` `인기` `✍️ 쓰기` `알림` `내정보`
- **글쓰기 탭은 중앙 플로팅 CTA** (당근마켓 스타일)
  - 둥근 원형 버튼, 아이콘만
  - 브랜드 컬러 강조
- iOS 하단 safe area 고려 (홈 인디케이터 공간 확보)

### 5-5. 댓글 블록 (Comment Block)
- 들여쓰기: 대댓글은 좌측 16pt 들여쓰기 + 세로 선 (threading 시각화)
- 댓글 헤더: 닉네임·업종·시간
- 본문
- 액션: 좋아요 / 답글
- **2단 이상 중첩 금지** (모바일에서 가독성 망함)

### 5-6. 카테고리 탭 (Category Tab)
- 가로 스크롤 칩 형태
- 활성: 브랜드 컬러 채움 + 흰 텍스트
- 비활성: 배경 투명 + 회색 텍스트

### 5-7. 플로팅 액션 버튼 (FAB)
- 하단 탭바 중앙 `쓰기` 버튼
- 지름 56pt, 브랜드 컬러, 그림자

## 6. 디자인 토큰 (Claude Design에게 제안 요청)

### 6-1. 컬러 팔레트 — **제안 요청**
브랜드 컬러 방향에 **정답이 없음**. 아래 3가지 방향성으로 각각 시안 주면 좋음:

- **방향 A — "따뜻한 신뢰"**: 네이버카페 그린 계열 or 당근 오렌지 계열
- **방향 B — "활기찬 유머"**: 블라인드 블루 or 청록
- **방향 C — "한국 전통 따뜻함"**: 황토색 or 진한 감색

**꼭 필요한 상태 컬러:**
- Primary (브랜드)
- Success (좋아요 활성)
- Warning (싫어요 활성, 단 과하지 않게)
- Danger (삭제/신고)
- Neutral 5단계 (배경/구분선/회색 텍스트)

### 6-2. 타이포그래피
- **한글 폰트: Pretendard (필수)**
- 영문/숫자: SF Pro (iOS) / Roboto (Android) 폴백

| 용도 | 크기 | weight |
|---|---|---|
| 제목 (H1) | 22pt | Bold |
| 카드 제목 | 17pt | SemiBold |
| 본문 | 15~16pt | Regular |
| 메타 (작성자/시간) | 13pt | Medium (회색) |
| 캡션 | 12pt | Regular |

**어르신 가독성 우선** — Thin/Light weight 사용 금지. Regular 이상.

### 6-3. 스페이싱 & 터치 타겟
- 8pt 그리드
- 터치 타겟 최소 **48×48pt** (Android 권장치 유지)
- 카드 내부 패딩: 16pt
- 섹션 간격: 16~24pt

### 6-4. 라운딩
- 카드: 0pt (블라인드 스타일) or 8pt (당근 스타일) — 제안 바람
- 버튼: 8~12pt
- 아바타/프로필 이미지: 원형 (50%)
- 배지: 4pt

### 6-5. 그림자
- 카드: 그림자 없음 (구분선으로만 분리)
- FAB: 중간 그림자 (elevation 4)
- 모달: 배경 오버레이 40% 블랙

## 7. React Native 기술 제약·고려사항

- iOS/Android **safe area** 둘 다 고려 (상단 노치, 하단 홈 인디케이터)
- iOS 스와이프로 뒤로가기 제스처 지원
- **다크모드는 v2로 미룸** — 현재는 라이트 모드만
- 이미지: `expo-image` 전제 (lazy loading, 캐싱)
- 폰트: `@expo-google-fonts/pretendard` or 번들링
- 애니메이션: `react-native-reanimated` 전제 (좋아요 인터랙션 등)
- **키보드 올라올 때 댓글 입력창이 키보드 위로 붙도록** (`KeyboardAvoidingView`)

## 8. 특수 요구사항

### 8-1. NPC(AI 페르소나) 고려
- 현재 서비스에는 AI NPC 21명이 실제 유저처럼 글·댓글을 작성한다.
- 유저 관점에서는 **NPC와 실제 유저를 구분할 수 없어야 한다**.
- → **"AI" 또는 "봇" 배지를 달지 말 것**
- → 닉네임 + 업종배지 표시는 유저/NPC 동일 처리

### 8-2. 이미지 포스트 비중
- 유머/짤방 카테고리는 이미지 포스트가 많다.
- 피드에서 이미지 포스트가 "튀어 보이지" 않고 자연스럽게 섞이도록.
- 썸네일 로딩 중 skeleton / blur placeholder 고려.

### 8-3. 어르신 친화 특수 배려
- 모든 CTA는 **텍스트 + 아이콘 병기**. 아이콘만 있는 버튼 금지.
- 에러 메시지는 "뭘 어떻게 하라"가 명확해야 함 ("오류가 발생했습니다" ❌ / "다시 시도해주세요" ✅).
- 확인/취소 다이얼로그에서 **파괴적 액션은 항상 회색·보조 위치**에.

## 11. 최종 산출물 요구사항 (Claude Design에게)

### 11-1. Figma 파일 구조 (또는 동등한 형태)
- **페이지 1**: 디자인 시스템 (컬러/타이포/컴포넌트)
- **페이지 2**: 주요 화면 (홈/상세/글쓰기/알림/프로필/로그인)
- **페이지 3**: 상태 변형 (빈 상태, 로딩, 에러)
- **페이지 4**: 다크모드 (v2 — 지금은 스킵 가능)
